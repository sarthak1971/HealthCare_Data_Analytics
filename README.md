# Healthcare Data Analytics Case Study

A comprehensive **SQL-based Healthcare Analytics Project** that analyzes patient behavior, doctor performance, hospital efficiency, revenue trends, and more using a relational database.

## 📋 Project Overview

This project demonstrates advanced SQL skills applied to a healthcare organization’s data. It includes revenue analysis, patient segmentation using **RFM modeling**, doctor performance evaluation, hospital benchmarking, payment analysis, cohort analysis, and more.

**Objective**: Generate actionable business insights to support data-driven decision-making in healthcare operations.

## ✨ Key Features

### Revenue Analysis
- Total revenue by treatment type
- Treatments per hospital
- Top 5 highest revenue treatments
- Average treatment cost by doctor specialization

### Patient Segmentation (RFM Analysis)
- Recency, Frequency, and Monetary value calculation
- RFM scoring using `NTILE()`
- Identification of **Top 10 High-Value Patients**

### Doctor Performance
- Revenue generated by each doctor
- Average patient rating per doctor
- Ranking doctors within each hospital

### Hospital Analysis
- Total revenue per hospital
- Hospital comparison within states
- Top-performing hospital in each state

### Additional Analyses
- Payment type distribution and contribution
- Cohort analysis & patient retention
- Date-based trends (monthly/yearly appointments & revenue)
- Patient summary and doctor performance **Views**

## 🗄️ Database Schema

The project uses the following tables:

- `patients`
- `doctors`
- `hospitals`
- `appointments`
- `treatments`
- `payments`
- `reviews`
- `diagnosis`
